# DSSAD 数据提取及分析软件 — 架构说明

> 版本：v1.0  
> 标准：GB 44497-2024《智能网联汽车 自动驾驶数据记录系统》  
> 技术栈：PyQt5 5.15 + OpenCV 4.11 + pyqtgraph 0.13 + openpyxl 3.1 + PyInstaller 6.13

---

## 1. 整体架构（分层）

```
┌─────────────────────────────────────────────────────────┐
│                    UI 层（PyQt5）                    │
│  LoginDialog  →  MainWindow → VideoPlayerWidget    │
│                              → ChartDockWidget      │
├─────────────────────────────────────────────────────────┤
│                   业务逻辑层                            │
│  DSSADReader（模拟 DoIP/UDS）                     │
│  LocalFileReader（本地文件）                        │
│  EventExporter（导出/Excel 转换）                   │
├─────────────────────────────────────────────────────────┤
│                   数据模型层                            │
│  models/data.py — EventType, VehicleInfo, Event    │
│                      EventDataPoint, DSSADPackage   │
├─────────────────────────────────────────────────────────┤
│                   工具层                                │
│  utils/helpers.py — 时间格式化、哈希、路径工具      │
│  utils/exporter.py — 事件导出、Excel 生成           │
└─────────────────────────────────────────────────────────┘
```

## 3. 数据模型（`src/models/data.py`）

### 3.1 枚举

| 枚举 | 值 |
|---|---|
| `EventType` | `LOCKED`（锁定事件）、`COLLISION`（碰撞事件）、`COLLISION_RISK`（碰撞风险事件）、`TIMESTAMP`（时间戳事件） |
| `DSSADType` | `TYPE_I`（I 型）、`TYPE_II`（II 型） |

### 3.2 数据类

```
VehicleInfo          — 车辆/设备基本参数（VIN、DSSAD 型号、存储统计等）
EventDataPoint      — 事件数据点（timestamp + values 字典）
Event               — 事件记录（核心业务对象，含数据点列表、视频文件列表等）
DSSADPackage       — DSSAD 导出包（vehicle_info + events 列表）
```

### 3.3 示例数据生成

- `generate_sample_vehicle_info()` — 生成模拟车辆信息
- `generate_sample_events(count)` — 生成 count 个随机示例事件

## 4. 核心模块说明

### 4.1 `src/core/reader.py` — 数据读取

| 类 | 职责 |
|---|---|
| `DSSADReader` | 模拟 DoIP/UDS 协议连接 DSSAD 设备；`connect()`、`read_events()`、`delete_events()` |
| `LocalFileReader` | 读取本地事件包目录（`read_package`）或单个 JSON 文件（`read_single_json`） |

> **注意**：`DSSADReader` 当前为模拟实现（内存数据），对接真实设备时需替换为真实 DoIP/UDS 通信逻辑。

### 4.2 `src/ui/login_dialog.py` — 登录

- `LoginDialog(QDialog)`：用户名/密码认证
- 默认账号：`admin` / `admin`
- 验证通过后发出 `login_success(username, password)` 信号

### 4.3 `src/ui/main_window.py` — 主窗口（核心）

#### 4.3.1 `MainWindow(QMainWindow)` — 8 大 UI 区域

```
┌──────────────────────────────────────────────────────────────────┐
│  菜单栏：文件（打开本地事件包 / 退出）| 帮助（操作手册 / 关于）│
├─────────────────────────┬────────────────────────────────────┤
│ ① 车辆/设备基本参数    │ ⑤ 视频播放区域                        │
│   （VIN、DSSAD 型号…）│   （OpenCV 播放器，摄像头切换，       │
│                         │    倍速、逐帧、时间轴、全屏）        │
├─────────────────────────┼────────────────────────────────────┤
│ ② 加载/读取事件文件    │ ⑥ 视频信息区域                        │
│    （连接 DSSAD /      │   （摄像头、当前时刻、记录起止时间、   │
```

#### 4.3.2 `VideoPlayerWidget(QWidget)` — 视频播放器

| 功能 | 实现方式 |
|---|---|
| 视频解码 | `cv2.VideoCapture` |
| 播放/暂停 | `QTimer` 定时触发 `_next_frame()` |
| 倍速 | 0.25x / 0.5x / 1.0x / 2.0x / 4.0x，动态调整 `QTimer` 间隔 |
| 逐帧 | `step_forward()` / `step_back()` 直接 seek |
| 时间轴 | `QSlider`，拖拽跳转 |
| 全屏 | `showFullScreen()` / `showNormal()` |
| 帧级同步 | 发射 `frame_changed(current_frame, total_frames)` 信号 |

#### 4.3.3 `ChartDockWidget(QWidget)` — 数据分析图表

- 使用 `pyqtgraph.PlotWidget` 绘制时序曲线
- X 轴：相对于事件起点的秒数
- Y 轴：用户选择的数据字段（速度、加速度、方向盘转角等）
- 以独立浮动窗口弹出，可同时打开多个图表

### 4.4 `src/utils/exporter.py` — 导出

| 方法 | 功能 |
|---|---|
| `export_events(events, vehicle_info)` | 将事件导出为本地目录结构，每个事件一个子目录，含 `metadata.json` + `data_points.json` |
| `convert_to_excel(events)` | 将所有事件汇总写入一个 `.xlsx` 文件，含表头样式 |

### 4.5 `src/utils/helpers.py` — 工具函数

| 函数 | 功能 |
|---|---|
| `format_time_ms(dt)` | 格式化 `datetime` 为毫秒精度字符串（UTC+8） |
| `parse_size(size_str)` | 解析存储容量字符串 |
| `compute_hash(data)` | 计算数据哈希（完整性校验） |
| `ensure_dir(path)` | 递归创建目录 |

## 5. 数据流向

```
┌──────────────┐   connect()    ┌──────────────────┐
│  DSSAD 设备  │──────────────→│  DSSADReader    │
│  (DoIP/UDS)  │               │  (模拟/真实)     │
└──────────────┘               └────────┬─────────┘
                                       │ read_events()
                                       ↓
┌──────────────┐   read_package()      ┌──────────────────┐
│  本地事件包   │──────────────→│  LocalFileReader │
│  (目录/JSON)  │               │                  │
└──────────────┘               └────────┬─────────┘
                                       │
                                       ↓
                              ┌────────────────┐
                              │  MainWindow    │
                              │  .events[]    │
                              └───┬────────────┘
                                  │ 选择事件
          ┌───────────────────────┼───────────────────────┐
          ↓                       ↓                       ↓
   VideoPlayerWidget      Event info labels      ChartDockWidget
   (加载视频文件)        (显示事件元数据)       (pyqtgraph 绘图)
          │
          ↓ frame_changed 信号
   _on_video_frame_changed()
          │
          ↓
   _sync_event_data_to_frame()
   (根据当前帧索引，同步显示对应数据点的实时值)
```

## 6. 关键设计决策

| 决策 | 原因 |
|---|---|
| 绝对导入（`from models.data import ...`） | PyInstaller 打包时相对导入会失败 |
| `sys.path.insert(0, SRC_DIR)` | 使 `main.py` 能正确找到 `src` 下的模块 |
| `--collect-submodules` | 确保 numpy / cv2 / pandas 的隐藏子模块被打包 |
| `--exclude-module` 排除 `.tests` | 减小打包体积（约 100MB） |
| OpenCV `cv2.VideoCapture` 视频播放 | 比 QMediaPlayer 更可控，支持逐帧精确 seek |
| pyqtgraph 而非 matplotlib | 实时数据刷新性能更好，适合时序曲线 |
| 示例数据内存生成 | 无需外部数据库，便于演示和测试 |

## 7. 构建与发布

### 7.1 开发运行

```bash
cd dssad-host
python main.py
```

### 7.2 打包 exe

```bash
python build.py
```

输出：`dist/DSSAD数据提取及分析软件/DSSAD数据提取及分析软件.exe`

### 7.3 构建参数说明

```
--onedir            → 输出为目录（含 _internal 依赖），便于携带 docs/sample_data
--windowed         → 无控制台窗口
--add-data docs     → 打包操作手册
--add-data sample_data → 打包示例数据
--collect-submodules → 收集所有子模块
--exclude-module    → 排除测试代码，减小体积
```
